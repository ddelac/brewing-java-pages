Brewing Java Usert Stories

Speedrunner:

  • As a speedrunner, I want a game that is not too easy nor too hard, and I want to see both score and time displayed in worldwide leaderboards. This game should also have an incentive to play, for example, a good ending or fun and creative gameplay.

  • Elaboration; By creative gameplay, I mean that the gameplay is not the same every time the player should not be able to play to the point where you already know what to expect. Every time you play, you should get a random order of challenges. As for a good incentive, I would like to have a good ending that makes you want to see that scene again and keep playing to improve and get there faster.

  • Constraints: None

  • Effort Estimation:

    • Databases: 4-8 hours
    • Random/uniqueness coding: 3-4 hours
    • Story: 2-4 hours
    • Gameplay: 2-4 hours
    • Implementation: 4-8 hours,
    • Total: 15-28 hours

  • Acceptance Test; Gameplay, the game should be played as much as possible, checking for uniqueness every time and checking for any bugs that may occur. At the end of every game double-check that score is saved and is accurate. Time and ranking should also be correct.

Competitive:

  • As a competitive player, I want to see my score compared against other players and work to move up the ranks.
  • Elaboration; Users should have the ability to see a scoreboard or rank board, where all the top users' scores are kept and updated. This should include the users' names and scores.
  • Constraints: None
  • Effort Estimation:
    • Learning Databases: 4-8 hours
    • Implementation: 4-8 hours
    • Total: 8-16 hours
  • Acceptance Test; Run through the game multiple times with different scores. Write each score down on a piece of paper. Put the written scores in order from most to least. Finally, check the written list against the rank/scoreboard.
